# Краткие теоретические сведения для пакета OpenMP

## Основные понятия
Распараллеливание в OpenMP выполняется с помощью явного указания компилятору путем использования директивы. Формат ввода:
    
    #pragma omp <имя директивы> [список параметров]

Параметры директив нужны для определения поведения программы при распараллеливании, а также для задания области видимости переменных и специальных действий в начале и конце куска кода.

### Список директив:

**1. Для определения параллельной области**

Используется модель, при которой в параллельных нитях исполняется одинаковый код. Выполнение программы всегда начинается в последовательном режиме – работает только одна главная нить. Области распараллеливания вычислений называются «параллельной областью», параллелизм начинается только при входе вычисления в параллельную область и завершается при выходе из нее.

Когда поток выполнения доходит до директивы parallel, создается набор (team) из N нитей. Входная нить этого набора (являющаяся главной нитью) всегда имеет номер 0. Код области действия директивы дублируется или разделяется между нитями для параллельного выполнения. В конце области обеспечивается синхронизация нитей – выполняется ожидание завершения вычислений всех нитей; далее все нити завершаются, и дальнейшие вычисления продолжает выполнять только главная нить. 

**2. Для распределения вычислений**

Эти директивы определяют то, как распределяются ресурсы вычислительных задач между потоками. Области распараллеливания могут вкладываться друг в друга.

Директива *for* распределяет по имеющимся нитям копии ассоциированного оператора цикла, каждая из которых выполняет некоторую часть от общего количества итераций.

Директива *sections* формирует нити из отдельных блоков кода, расположенных в исходной программе последовательно друг за другом. Перед каждым преобразуемым в нить блоком указывается вспомогательная директива *section*.

Есть *shared* (видны всем потокам) и *private* (свои для каждого потока) переменные.

Редукция - функция высшего порядка, которая производитпреобразование структуры данных к единственному атомарному значению при помощи заданной функции.


**3. Для управления ходом вычислений**

Директивы управления ходом вычислений предназначены для изменения типа процесса вычисления, определенного параллельной областью или вышестоящей директивой распределения вычислений. В основном они, в отличие от директив распределения вычислений, предназначены для ограничения применения параллелизма в своей области действия.

**4. Для синхронизации**

Директивы синхронизации предназначены для управления взаимодействием параллельных нитей между собой в ходе параллельного вычисления. OpenMP в определенных случаях позволяет явно определить тот или иной способ синхронизации нитей. Atomic, barrier, critical - всё как у всех.

## Функции и переменные окружения

Есть функции, которые повзоляют получить системные данные об области распараллеливания (например, количество потоков) и для замера времени. Для всё этого используются системные переменные.
