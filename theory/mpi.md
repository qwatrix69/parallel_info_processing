# Краткие теоретические сведения для пакета MPI

## Основные понятия
В пакете MPI есть понятие **области связи**. Для коммуникации при параллельном взаимодействии все процессы должны находиться в одной области связи. Внутри области связи процессы имеют свою нумерацию.

**Коммуникатор** - это описатель области связи. Предоставляется пользователю для ограничения области связи при работе с процессами. У одной области связи может быть несколько коммуникаторов. Базовый коммуникатор *MPI_COMM_WORLD* - создается автоматически, описывает стартовую область связи для объединения всех процессов приложения.

Функции в mpi по принципу исполнения делятся на **блокирующие** и **неблокирующие**. Блокирующие - **синхронные** и **асинхронные**. По области действия на **локальные, точка-точка, и групповые**. Локальные - без обмена данными (инфо, системные, ждущие), точка-точка - два экземпляра программы, коллективные - между всеми процессорами коммуникатора.

Функции MPI для работы с входными и выходными параметрами используют *нетипизированные указатели типа void* *, поэтому функции MPI по указателю никак не могут определить, какой тип данных на самом деле используется. Поэтому почти во всех функциях передаются параметры, определяющие тип данных для передачи. Они практически полностью совпадают с типами языка Си.

## Функции MPI
Есть несколько категорий функции MPI, расписывать все смысла нет, опишу основные принципы.

**1. Общие функции** 
Работа с окружением, получение служебной информации, работа с замером времени - функционал этих методов очевиден из названия.

**2. Функции точка-точка для пересылки**
Отправка и прием, одновременная отправка и прием - типовые функции с указанием источника, приемника, размера, типа данных и коммуникатора, есть механизм отслеживания статуса.

**3. Функции коллективного обмена**
Должны вызываться либо всеми процессами внутри области связи, либо никакими. Для разграничения и разделения процессов можно создать допонительный коммуникатор.

Рассылка сообщений, рассылка частей сообщения, сборка по частям, обмен частями сообщений и так далее - разбиение данных от одного процесса к нескольким и наоборот. 

Есть функции для применения операций, в одном из параметров указывается тип применяемой операции к функции обмена. Есть функции, возвращающие данные в процессы, части данных.

Для синхронизации используется типовой барьер - при вызове функции синхронизации работа всех процессов блокируется, пока они не дойдут до того же места в коде.
